# Recursion 0.1.0-pre-alpha.1

Recursion `0.1.0-pre-alpha.1` is the first documented pre-alpha checkpoint for the SillyTavern extension. The release is private pre-alpha and uses the V1 contract described in the current spec set.

## Current Scope

Recursion is a current-scene prompt compiler. It observes the active SillyTavern chat, uses Utility-led structured analysis to maintain a bounded scene deck, selects a compact turn hand, optionally uses Reasoner for difficult composition, and installs a bounded prompt packet when Auto or Manual mode is active. Pipeline mode is separate from Auto/Manual: Standard runs the full foreground path on send, while Rapid warms provider-generated scene guidance in the background and uses a shorter foreground Utility delta or fast-start pack.

The pre-alpha boundary is intentionally narrow. Recursion does not own durable memory, lore databases, transcript summarization, vector recall, campaign saves, character databases, or user-authored card editing.

## Implemented V1 Surfaces

- Extension manifest and SillyTavern entrypoint: `manifest.json`, `src/extension/index.js`
- Runtime modules for settings, storage, activity, providers, cards, prompt packets, runtime orchestration, UI, and SillyTavern host adapters.
- Recursion Bar, icon-only Standard/Rapid Pipeline control, icon-only Auto/Manual mode control, Cards scope selector, Hero Pixel Array progress menu, options/settings menu, Last Brief dropdown, Full Viewer, autosaving settings, and provider controls as the current UI contract.
- Standard foreground pipeline and Rapid warm/delta/fast-start pipeline, including exact-source warm artifacts, Utility-only Rapid foreground roles, optional hedged Rapid Utility calls, no local Rapid substitute guidance, and Standard escalation for invalid Rapid output or mandatory gaps.
- Utility provider lane as the default structured-work lane.
- Optional Reasoner provider lane with Utility fallback.
- Reasoning Level route policy for Low/Medium/High/Ultra lane depth and card pressure.
- Behavior policy module for Strength, Min/Max Cards, Focus, and Prompt Footprint backend effects.
- Provider source control for current host model, host connection profiles, and OpenAI-compatible endpoints with read-only model discovery.
- Machine-JSON provider request metadata so host connection profile calls can request the expected Recursion response schema without host preset/instruct wrapping.
- Stricter structured-output contract: Arbiter prompts and retry prompts restate required `schema`/`snapshotHash`, and provider calls use a 120 second default timeout.
- Prompt packet composition and Recursion-owned SillyTavern prompt block installation/clear behavior.
- Chat/source/generation-stop cleanup that aborts stale Recursion work, clears Recursion prompt keys, and reports player Stop as skipped progress.
- Bounded scene cache, run journal, sanitized diagnostics, and session-only direct endpoint secret handling.
- Deterministic tests plus guarded Playwright/live smoke support.

## Live Smoke Guardrails

Live SillyTavern checks are opt-in and must use dedicated users such as `recursion-soak-a`, `recursion-soak-b`, or `recursion-soak-c`. Live scripts must reject `default-user` before login, browser navigation, storage probes, chat mutation, prompt injection, or provider calls.

Live smoke is for guarded evidence only. Normal completion of this documentation checkpoint does not require mutating a live SillyTavern profile.
