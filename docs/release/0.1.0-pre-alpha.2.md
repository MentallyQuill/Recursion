# Recursion 0.1.0-pre-alpha.2

Recursion `0.1.0-pre-alpha.2` is the Dialogue Enhancement pre-alpha checkpoint. It keeps the V1 current-scene prompt compiler scope from `0.1.0-pre-alpha.1` and adds the broader post-generation Enhancements surface.

## Release Highlights

- Replaced the Prose Enhancement-only settings/UI contract with `Enhancements`.
- Added targets: `Off`, `Prose`, `Dialogue`, and `Prose + Dialogue`.
- Added apply modes: `As Swipe` and `Replace`.
- Added `dialogueEnhancer` as a Utility role with schema `recursion.dialogueEnhancer.v1`.
- Added Dialogue Enhancement prompting and validation for echoing, forced questions, over-technical dialogue, unsupported tsundere deflection, attraction cliches, and weak subtext.
- Runs `Prose + Dialogue` as Dialogue first, then Prose, with one final applied result.
- Kept post-generation enhancement before Rapid warm so Rapid sees the final selected assistant text.
- Added guarded Playwright proof for Dialogue and Prose + Dialogue against a real SillyTavern Utility provider.

## Verification Commands

Maintained local gate:

```powershell
npm.cmd test
```

Live feature proof used for this checkpoint:

```powershell
node tools\scripts\prove-live-prose-enhancement.mjs
node tools\scripts\prove-live-enhancements.mjs
```

The live proof must run against a dedicated `recursion-soak-*` user. For this checkpoint it verified:

- Utility provider live call succeeds.
- Prose Enhancement `Off`, `As Swipe`, and `Replace` still work.
- Dialogue Enhancement `As Swipe` creates and selects an enhanced sibling swipe.
- Prose + Dialogue `Replace` runs Dialogue then Prose and replaces the active assistant text.
- The served SillyTavern UI exposes one Enhancements button with target rows `Off`, `Prose`, `Dialogue`, `Prose + Dialogue` and apply rows `As Swipe`, `Replace`.
