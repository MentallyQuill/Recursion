# Recursion 0.1.0-pre-alpha.3

Recursion `0.1.0-pre-alpha.3` tightens the Dialogue Enhancement checkpoint after live SG-1 testing showed High Reasoning Level enhancement calls silently running through Utility when the Reasoner provider was disabled.

## Release Highlights

- Changed Enhancements routing so Low and Medium use Utility, while High and Ultra request Reasoner directly.
- Preserved the existing reveal-original safety path when the selected enhancement provider lane is unavailable, fails, or returns invalid output.
- Updated operator/provider docs so Enhancements no longer imply hidden Utility fallback at High or Ultra.
- Kept the Dialogue Enhancement and Prose + Dialogue UI fixes from `0.1.0-pre-alpha.2`.

## Verification Commands

Focused local checks:

```powershell
node tools\scripts\test-runtime.mjs
node tools\scripts\test-providers.mjs
node tools\scripts\test-ui.mjs
git diff --check
```

Live install verification copied the updated files into the `default-user`, `recursion-soak-a`, `recursion-soak-b`, and `recursion-soak-ui` SillyTavern extension profiles and confirmed the served manifest reports `0.1.0-pre-alpha.3`.
