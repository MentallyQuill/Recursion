# Recursion `0.1.0-pre-alpha.6`

`0.1.0-pre-alpha.6` makes the card system a complete operator-facing feature surface and carries the first unified editorial recovery pipeline.

## Highlights

- Added editable custom card decks with categories, authored cards, duplication, deletion, and deterministic ordering.
- Added card selection states: `off`, `active`, and `priority`, including bulk actions and Max Cards handling.
- Added card-scope controls for Auto and Manual operation, family/sub-item focus, strict whitelist behavior, and selection summaries.
- Added card authoring assistance with bounded provider output and an explicit commit boundary.
- Expanded card inspection through Last Brief and the Full Viewer, including hand contents, omissions, origins, and metadata.
- Added mobile-aware card and deck interactions, including touch-friendly state controls and drag handles.
- Updated the card design, technical manuals, operator manual, first-run workflow, README, Mermaid diagrams, and documentation render inventory.

## Editorial and runtime additions

- Replaced separate prose/dialogue rewrites with one card-aware Generation Review and Enhancement contract supporting `Off`, `Repair`, `Recompose`, and `Redirect`.
- Added evidence-bounded Redirect diagnosis, transformation, verification, and effectiveness judging, with no-write behavior when the trajectory cannot be safely replaced.
- Added shared structured-output recovery with one correction budget, local malformed-JSON repair, raw machine-JSON preservation at the host boundary, and explicit `partial-failed`/provider/validation status.
- Added exact-source Rapid reuse, swipe-aware packet reuse, fresh-next-generation bypass, Fused bundle repair, Reasoner routing, and inline failure reasons.
- Added deterministic and live proof commands for card UI, card progress, editorial UI, Enhancement runs, and strict dedicated-user host verification.

## Card model

Cards remain scene-local reasoning artifacts, not durable memory. The bundled catalog covers scene frame, active cast, character motivation, dialogue relationship, social subtext, scene constraints, knowledge and secrets, clocks and consequences, environment and affordances, possessions and items, and open threads.

The active deck is the operator-controlled source of card scope and ordering. Recursion selects a bounded turn hand from eligible cards, then uses that hand as evidence for the prompt packet. A card can exist in the deck without appearing in the current hand.

## Verification posture

This remains a pre-alpha release. Card behavior is covered by the focused card, UI, storage, prompt, runtime, and live-proof checks. Documentation renders must come from the sterile live-host render profile and must not expose provider secrets, raw prompts, raw responses, hidden reasoning, or private transcripts.
