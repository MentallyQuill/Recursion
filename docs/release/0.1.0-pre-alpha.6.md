# Recursion `0.1.0-pre-alpha.6`

`0.1.0-pre-alpha.6` makes the card system a complete operator-facing feature surface and adds the first V1 Post-process Cards workflow for revising completed responses through SillyTavern's native host generation path.

## Highlights

- Added editable custom card decks with categories, authored cards, duplication, deletion, and deterministic ordering.
- Added card selection states: `off`, `active`, and `priority`, including bulk actions and Max Cards handling.
- Added card-scope controls for Auto and Manual operation, family/sub-item focus, strict whitelist behavior, and selection summaries.
- Added card authoring assistance with bounded provider output and an explicit commit boundary.
- Expanded card inspection through Last Brief and the Full Viewer, including hand contents, omissions, origins, and metadata.
- Added mobile-aware card and deck interactions, including touch-friendly state controls and drag handles.
- Added independent Pre-process Cards and Post-process Cards surfaces with shared compact deck, category, card, description, disclosure, and drag behavior.
- Updated the card design, technical manuals, operator manual, first-run workflow, README, Mermaid diagrams, and documentation render inventory.

## Post-process and runtime additions

- Added an independent ordered Post-process deck with binary card `On` / `Off` state, frozen completed-response evidence, and configurable `Unified` or `Progressive` flow.
- Added structured guidance synthesis through the routed Utility or Reasoner lane while keeping SillyTavern's native quiet-generation path as the sole prose writer.
- Added `As Swipe` and `Replace` settlement, same-lane guidance retries, host-writer retry with packet reuse, stale-operation rejection, cancellation, and fail-soft Progressive partial settlement.
- Added shared structured-output recovery with one correction budget, local malformed-JSON repair, raw machine-JSON preservation at the host boundary, and explicit `partial-failed`/provider/validation status.
- Added exact-source Rapid reuse, prepared packet-and-hand swipe reuse, fresh-next-generation bypass, Fused bundle repair, explicit provider routing, and inline failure reasons.
- Hardened host lifecycle ownership for terminal generation events, duplicate finalization, native Stop behavior, control unlocking, and internal quiet-generation interception.
- Added deterministic and live proof commands for card UI, card progress, Post-process UI, Post-process host settlement, swipe reuse, and strict dedicated-user host verification.

## Card model

Cards remain scene-local reasoning artifacts, not durable memory. The bundled catalog covers scene frame, active cast, character motivation, dialogue relationship, social subtext, scene constraints, knowledge and secrets, clocks and consequences, environment and affordances, possessions and items, and open threads.

The active deck is the operator-controlled source of card scope and ordering. Recursion selects a bounded turn hand from eligible cards, then uses that hand as evidence for the prompt packet. A card can exist in the deck without appearing in the current hand.

## Verification posture

This remains a pre-alpha release. Card behavior is covered by the focused card, UI, storage, prompt, runtime, host, Post-process, and live-proof checks. Current proof entry points include `npm.cmd test`, `npm.cmd run prove:card-system-ui`, `npm.cmd run prove:post-process-ui`, `npm.cmd run prove:post-process-live`, and `npm.cmd run prove:swipe-reuse-live`. Documentation renders must come from the sterile live-host render profile and must not expose provider secrets, raw prompts, raw responses, hidden reasoning, or private transcripts.
