# Recursion Cost Research

This document summarizes the current token and call-cost shape of Recursion. It is a planning and operator reference, not a billing engine. Recursion does not currently store price-per-token, token multipliers, provider billing rates, or monthly quota settings in runtime configuration.

Cost in this document means one of three things:

- provider calls Recursion makes before or after the host generation;
- tokens sent to, returned by, or injected into model calls;
- externally billed budget tokens after a provider-specific multiplier is applied outside Recursion.

## Current Runtime Position

Recursion uses a two-lane provider model:

| Lane | Required | Typical cost role |
| --- | --- | --- |
| Utility | Yes | Fast, cheap, JSON-first operational work: Arbiter, card generation, Fused bundles at Low/Medium, Guidance Composer, Rapid foreground delta, provider tests, and Enhancements. |
| Reasoner | No | Optional synthesis lane selected by Reasoning Level when enabled and healthy. It is used for `reasonerComposer`, and for Arbiter/card/Fused work at higher reasoning levels. |

Recursion does not have a Writer lane. The final story prose is generated by SillyTavern's normal host model after Recursion installs its prompt packet.

Each lane can use one provider source:

- `host-current-model`
- `host-connection-profile`
- `openai-compatible`

OpenAI-compatible direct keys are session-only. Provider prompts, provider responses, API keys, hidden reasoning, full transcript text, and full prompt packets must not enter saved settings, journals, diagnostics, browser storage, or test artifacts.

## Billing Configuration Boundary

No runtime monetary billing configuration exists today. Source and docs use token-cost language only for planning:

| Concept | Meaning |
| --- | --- |
| `tokenCost` on cards | A card prompt-footprint estimate, not a provider bill. |
| `tokenEstimate` in catalog entries | A planning estimate used for hand and budget shaping. |
| Footprint section budgets | Character caps for Guidance, Card Evidence, and Guardrails. |
| Provider `maxTokens` | Per-lane response cap, default `8192`. |
| External multipliers | Provider- or subscription-specific accounting outside Recursion. |

If a provider bills a model at 2x, Recursion does not know that natively. Operators should apply that multiplier externally when estimating budget consumption.

## Default Settings That Affect Cost

| Setting | Default | Cost impact |
| --- | ---: | --- |
| `pipelineMode` | `standard` | Full foreground Arbiter, card, compose, and install path. |
| `reasoningLevel` | `medium` | Utility handles Arbiter, cards, and Fused bundles by default; Reasoner may handle composition when enabled and healthy. |
| `minCards` / `maxCards` | `3` / `10` | Shape card job pressure; Medium/High use the normalized midpoint as a cap. |
| `promptFootprint` | `normal` | Controls final injected packet size budget. |
| `providerVisibleMessages` | `12` | Recent visible messages sent to Recursion provider calls. |
| `sourceWindowMessages` | `20` | Recent visible messages considered for source freshness and evidence windows. |
| `sourceWindowCharacters` | `12000` | Character cap for the source freshness window. |
| `enhancements.target` / `enhancements.applyMode` | `off` / `as-swipe` | Adds post-generation Utility rewrite calls when target is `prose`, `dialogue`, or `prose-dialogue`; combined target runs two calls. |
| Utility `maxTokens` | `8192` | Response cap for Utility calls. |
| Reasoner `maxTokens` | `8192` | Response cap for Reasoner calls. |

## Pipeline Call Shape

### Standard

Standard is the reference foreground path.

| Stage | Call count | Lane |
| --- | ---: | --- |
| Arbiter | 1 | Utility at Low/Medium; Reasoner at High/Ultra when healthy. |
| Card generation | 0 to selected budget | Utility at Low/Medium; mixed or Reasoner-heavy at High/Ultra when healthy. |
| Guidance Composer | 1 when composing a packet | Utility. |
| Reasoner Composer | 0 or 1 | Reasoner when Medium+ policy and provider health allow it. |
| Enhancements | 0 to 2 after host generation | Utility, only when enabled. |

Card generation can be one router batch when batching is available, or multiple individual role calls. Cache reuse and `skip` plans reduce or remove downstream calls.

### Fused

Fused keeps the normal foreground lifecycle, but replaces many card calls with one `fusedCardBundle` call.

| Stage | Call count | Lane |
| --- | ---: | --- |
| Arbiter | 1 | Same policy as Standard. |
| Fused card bundle | 1 primary call | Utility at Low/Medium; Reasoner at High/Ultra when healthy. |
| Fused repair | 0 to missing/invalid family count | Standard card calls for damaged siblings only. |
| Guidance Composer | 1 when composing a packet | Utility. |
| Reasoner Composer | 0 or 1 | Reasoner when Medium+ policy and provider health allow it. |
| Enhancements | 0 to 2 after host generation | Utility, only when enabled. |

Fused is usually cheaper than Standard when the model returns trustworthy multi-card JSON because it collapses card setup overhead into one larger structured request.

### Rapid

Rapid moves most scene work into a background warm pass and keeps the foreground send path small when the warm artifact is exact-source and valid.

| Stage | Call count | Lane |
| --- | ---: | --- |
| Background warm Arbiter | 1 before the next turn | Same policy as Standard. |
| Background warm card generation | Standard card pipeline | Same policy as Standard. |
| Background warm Guidance Composer | 1 | Utility. |
| Foreground Rapid Turn Delta | 1, with optional hedge | Utility. |
| Foreground packet install | 0 model calls | Uses warm guidance plus delta. |
| Escalation to Standard | Standard call shape | Happens when warm artifact is missing, stale, invalid, empty, or declares a mandatory gap. |
| Enhancements | 0 to 2 after host generation | Utility, only when enabled. |

Rapid can reduce send-time cost and latency on warm hits, but warm work still consumes provider calls. A warm miss or mandatory gap can add the Rapid delta attempt plus a full Standard path for that same pending user message.

## Final Prompt Packet Size

Recursion also changes the host generation cost by injecting Guidance, Card Evidence, and Guardrails into SillyTavern's prompt. The final packet is bounded by Prompt Footprint section budgets in characters:

| Footprint | Guidance | Card Evidence | Guardrails | Approx total chars | Approx tokens |
| --- | ---: | ---: | ---: | ---: | ---: |
| Compact | 900 | 12,000 | 700 | 13,600 | 3,400 |
| Normal | 1,800 | 30,000 | 900 | 32,700 | 8,200 |
| Rich | 3,200 | 60,000 | 1,200 | 64,400 | 16,100 |

These packet tokens are not a separate Recursion provider call after install. They are extra input context for the host story-generation request.

## Example Estimate: Medium Reasoning, Normal Footprint

This example assumes:

- Utility lane: a 1x utility-class model.
- Reasoner lane: a 2x model such as GLM-5.2 through a SillyTavern connection profile.
- Host generation: the same 2x model.
- Reasoning Level: Medium.
- Pipeline: Standard.
- Prompt Footprint: Normal.
- Enhancements: Off.
- Typical source window: 15-20 visible messages, with provider calls still capped by `providerVisibleMessages`.

At Medium, Arbiter and card generation stay on Utility, `guidanceComposer` stays on Utility, and `reasonerComposer` may add a Reasoner synthesis patch when the Reasoner lane is enabled and healthy.

| Work | Lane | Raw tokens | Multiplier | Budget tokens |
| --- | --- | ---: | ---: | ---: |
| Arbiter | Utility | ~6,000 | 1x | ~6,000 |
| Card generation, roughly 6 cards | Utility | ~14,500 | 1x | ~14,500 |
| Guidance Composer | Utility | ~6,500 | 1x | ~6,500 |
| Reasoner Composer | Reasoner | ~5,500 | 2x | ~11,000 |
| Host generation with normal packet | Host | ~20,200 | 2x | ~40,400 |
| Approx full turn | Mixed | ~52,700 | mixed | ~78,400 |

Without Recursion, the same host generation might consume roughly `~11,200` raw host tokens, or `~22,400` budget tokens at 2x. In that scenario, Recursion's incremental cost is roughly `~56,000` budget tokens per full Standard turn before cache reuse, retries, provider hidden reasoning, or prose enhancement.

If a subscription supplies `60,000,000` monthly budget tokens for `$12`, then each 1,000 budget tokens is roughly `$0.0002`. Under the example above:

| Metric | Approx value |
| --- | ---: |
| Incremental Recursion cost per Standard turn | ~56,000 budget tokens |
| Incremental dollar cost per Standard turn | ~$0.011-$0.015, roughly 1-1.5 cents |
| Full turn cost with Recursion | ~78,400 budget tokens |
| Full dollar cost with Recursion | ~$0.016 |
| Monthly turns if every token is spent on this pattern | ~765 turns |

Bottom line: with the medium-reasoning Standard setup above, Recursion adds about 1-1.5 cents per turn on top of normal SillyTavern generation cost.

Treat those values as order-of-magnitude planning numbers, not a guarantee. Hidden thinking tokens on reasoning models can materially increase billed usage even when visible output is small.

## Cost Levers

Use these controls first when tuning cost:

| Lever | Effect |
| --- | --- |
| Use Low or Medium Reasoning Level | Keeps more work on Utility and reduces Reasoner usage. |
| Use Fused with a strong structured-output model | Reduces card-call overhead when bundle JSON is reliable. |
| Use Rapid after the scene is warm | Reduces foreground send-time calls on warm hits. |
| Lower `maxCards` and `minCards` | Reduces card generation and selected evidence. |
| Use Compact Prompt Footprint | Reduces final host prompt injection size. |
| Lower provider-visible and source-window limits | Reduces provider input tokens for long chats. |
| Keep Enhancements off | Avoids large post-generation Utility rewrite calls. |
| Prefer cache reuse for stable scenes | Lets the Arbiter skip new card generation when safe. |

## Caveats

- Provider hidden reasoning may be billed but not visible in Recursion output.
- Retries, Rapid hedging, Fused repair, and Rapid Standard escalation add calls.
- Host presets, character context, lorebooks, and other SillyTavern extensions can dominate final host-generation cost; Recursion only controls its own injected packet.
- A healthy Reasoner lane is optional. If unavailable, Recursion composes through Utility and records fallback metadata.
- Exact monthly cost depends on the external provider's accounting rules. Recursion currently exposes token-shaping controls, not provider billing controls.
